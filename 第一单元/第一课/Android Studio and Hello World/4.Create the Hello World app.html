2.3浏览Gradle Scripts文件夹

Android Studio中的Gradle构建系统可轻松将外部二进制文件或其他库模块作为依赖项包含在构建中。

首次创建应用程序项目时，将显示“项目”>“ Android”窗格，并展开了“ Gradle Scripts”文件夹，如下所示。

请按照以下步骤探索Gradle系统：

  1. 如果“ Gradle Scripts”文件夹没有展开，请单击三角形以将其展开。

该文件夹包含构建系统所需的所有文件。

  1. 查找build.gradle（Project：HelloWorld）文件。

在这里，您将找到构成项目的所有模块所共有的配置选项。每个Android Studio项目都包含一个顶级Gradle构建文件。在大多数情况下，您无需对此文件进行任何更改，但是了解其内容仍然很有用。

默认情况下，顶级构建文件使用buildscript块定义项目中所有模块通用的Gradle存储库和依赖项。如果您的依赖项不是本地库或文件树，则Gradle会在此文件的存储库块中指定的任何在线存储库中查找文件。默认情况下，新的Android Studio项目将JCenter和Google（包括Google Maven存储库）声明为存储库位置： 

            allprojects {
                repositories {
                    google()
                    jcenter()
                }
            }
  1. 查找build.gradle（Module：app）文件。

除了项目级别的build.gradle文件之外，每个模块都有自己的build.gradle文件，该文件可让您为每个特定模块配置构建设置（HelloWorld应用程序只有一个模块）。配置这些构建设置可让您提供自定义打包选项，例如其他构建类型和产品口味。您还可以覆盖AndroidManifest.xml文件或顶级build.gradle文件中的设置。

此文件通常是更改应用程序级配置（例如在“依赖项”部分中声明依赖项）时要编辑的文件。您可以使用几种不同的依赖项配置之一来声明库依赖项。每个依赖项配置为Gradle提供有关如何使用该库的不同说明。例如，语句实现fileTree（dir：'libs'，include：['* .jar']）在libs目录中添加所有“ .jar”文件的依赖项。

以下是HelloWorld应用程序的build.gradle（Module：app）文件： 

    apply plugin: 'com.android.application'

    android {
        compileSdkVersion 26
        defaultConfig {
            applicationId "com.example.android.helloworld"
            minSdkVersion 15
            targetSdkVersion 26
            versionCode 1
            versionName "1.0"
            testInstrumentationRunner 
                       "android.support.test.runner.AndroidJUnitRunner"
        }
        buildTypes {
            release {
                minifyEnabled false
                proguardFiles 
                         getDefaultProguardFile('proguard-android.txt'), 
                         'proguard-rules.pro'
            }
        }
    }
    
    dependencies {
        implementation fileTree(dir: 'libs', include: ['*.jar'])
        implementation 'com.android.support:appcompat-v7:26.1.0'
        implementation 'com.android.support.constraint:constraint-layout:1.0.2'
        testImplementation 'junit:junit:4.12'
        androidTestImplementation 'com.android.support.test:runner:1.0.1'
        androidTestImplementation  'com.android.support.test.espresso:espresso-core:3.0.1'
    }
  1. 单击三角形以关闭Gradle脚本。

该应用程序的所有代码和资源都位于app和res文件夹中。

  1. 展开app文件夹，java文件夹和com.example.android.helloworld文件夹，以查看MainActivity java文件。双击该文件可在代码编辑器中将其打开。 应用文件夹

Java文件夹在三个子文件夹中包含Java类文件，如上图所示。 com.example.hello.helloworld（或您指定的域名）文件夹包含应用程序包的所有文件。其他两个文件夹用于测试，并在另一堂课中进行介绍。对于Hello World应用程序，只有一个软件包，其中包含MainActivity.java。通常，用户看到的第一个Activity（屏幕）的名称也将初始化应用程序级资源，该名称通常称为MainActivity（在Project> Android窗格中省略了文件扩展名）。

清单文件夹中包含的文件可向Android系统提供有关您的应用程序的基本信息，系统必须具备该文件才能运行任何应用程序代码。

AndroidManifest.xml文件描述了Android应用程序的所有组件。应用程序的所有组件（例如每个Activity）都必须在此XML文件中声明。在其他课程中，您将修改此文件以添加功能和功能许可。有关介绍，请 参阅AppManifest概述。
